## ðŸ“Š Selection by Project Size

### Small Projects (<3 months, <5 people, <$100K)

**Essential (4 templates):**
- Project Charter - Define scope
- Simple Schedule - Track timeline
- Weekly Status Report - Communicate progress
- Issues Log - Track problems

**Nice-to-Have (2 templates):**
- Risk Register - If any significant risks
- Stakeholder Analysis - If multiple stakeholders

**Skip:**
- WBS (too detailed)
- Vendor Management (if no vendors)
- Full RAID Log (Issues Log sufficient)

**Total Time:** 6-8 hours setup

---

### Medium Projects (3-9 months, 5-15 people, $100K-$500K)

**Essential (8 templates):**
- Project Charter
- Stakeholder Analysis
- RACI Matrix
- WBS
- Project Schedule
- Budget Tracking
- Weekly Status Report
- Issues Log

**Recommended (4 templates):**
- Risk Register
- Communication Plan
- Change Request
- Lessons Learned

**Optional (3 templates):**
- Status Dashboard
- Decision Log
- Meeting Notes

**Total Time:** 20-30 hours setup

---

### Large Projects (9+ months, 15+ people, $500K+)

**Use Most Templates:**

**Core Set (15 templates):**
- All Essential and Recommended from Medium
- Plus: Dependency Matrix, RAID Log, Test Plan, Quality Checklist

**Add Based on Needs:**
- Vendor Management (if vendors)
- BRD (if complex requirements)
- RFP (if procurement)
- Training Plan (if system change)
- Go-Live Checklist (if deployment)
- Transition Plan (if handoff)

**Total Time:** 40-60 hours setup

## ðŸš¨ Red Flags: When You're Using Too Many Templates

**Warning Signs:**
- Spending more time on documentation than actual work
- Templates not being updated (stale)
- Team complaining about bureaucracy
- Templates with no clear purpose
- Duplicating information across templates

**Solution:**
- Audit your templates quarterly
- Remove or simplify unused templates
- Combine related templates (e.g., use RAID Log instead of separate Risk + Issues)
- Focus on templates that add value

## ðŸ’¡ Pro Tips

1. **Start Minimal** - Begin with 3-5 templates, add more only if needed
2. **Combine Templates** - Use RAID Log instead of 4 separate logs
3. **Tailor to Audience** - Executive stakeholders need Status Dashboard, not detailed logs
4. **Iterate** - Adjust template selection based on what works
5. **Ask Your Team** - Which templates add value? Which are bureaucracy?
